# SAFER K-64 Lab — документация проекта

Учебный проект: **блочный шифр** на 8 байт (в духе SAFER K-64), **GUI** (PySide6) и **набор демонстрационных атак** для курса по криптографии.

> **Не для реальной защиты данных.** Алгоритм и атаки упрощены для обучения.

Краткий запуск: `pip install -r requirements.txt` из корня репозитория, затем `python main.py` или `python run_gui.py`.

---

## 1. Возможности

- **Шифрование (вкладка «Шифрование»):** текст UTF-8 ↔ шифр в hex; ключ **16 hex-символов** (ровно 8 байт); число **раундов 1–4**; генерация случайного и «слабого» ключа.
- **Файлы (вкладка «Файлы»):** шифрование и расшифрование **сырых байтов** файла с дополнением **PKCS#7** до длины, кратной 8.
- **Криптоанализ:** несколько атак по очереди, текстовый лог, **графики** (время, успех/кандидаты, гистограмма байт шифра), полноэкранный режим для графиков.

## 2. Установка и запуск

py -3.12 -m venv .venv

## 4. Руководство пользователя

### 4.1. Вкладка «Шифрование»

1. Введите **открытый текст** (кодируется в UTF-8).
2. Укажите **ключ**: ровно **16 символов** `0-9a-f` (8 байт).
3. Выберите **число раундов** (1–4).
4. **Зашифровать** — в поле шифра появится **hex** (длина строки чётная; число байт кратно 8).
5. **Расшифровать** — вставьте hex шифра; ключ и раунды должны совпадать с шифрованием.

Дополнительные поля (вторая пара блоков P2/C2 в hex и т.д.) используются для переноса данных на вкладку «Криптоанализ».

### 4.2. Вкладка «Криптоанализ»

1. Заполните **P** (открытый текст) и **C** (шифр в hex) или нажмите **«Взять с шифрования»**.
2. При необходимости задайте **P2 и C2** (каждый — **16 hex-символов**, т.е. 8 байт), частично известный ключ, ключ оракула для линейного анализа — ориентируйтесь на подписи полей и сообщения атак.
3. Отметьте одну или несколько **атак** в списке.
4. **«Запустить»** — атаки выполняются **последовательно**; ход отображается в **логе** (верхняя правая область).
5. **Графики** (нижняя правая область): сравнение времени, успех и «вес» кандидатов, гистограмма байт поля C. Кнопка **«На весь экран»** открывает только блок графиков; **Esc** закрывает полноэкранный режим.

### 4.3. Вкладка «Файлы»

Работа идёт с **бинарным содержимым** файлов (не «как текст в кодировке редактора»).

**Шифрование файла**

- **Вход:** любой файл (его байты читаются как есть).
- **Выход:** файл, в который записывается **шифр** (файл перезаписывается).

**Расшифрование файла**

- **Вход:** только файл **шифротекста**, полученный этой же программой при шифровании (тот же ключ и то же число раундов).
- **Выход:** файл, куда будет записан **расшифрованный** поток байтов (после снятия PKCS#7, если хвост валиден).

**Типичная ошибка:** при расшифровке снова указать во «Вход» **исходный текстовый** файл. Его длина обычно **не кратна 8 байт**, появляется сообщение о том, что длина не кратна 8 — это **ожидаемо**: такой файл не является выходом блочного шифра. Нужно указать **файл с шифром** (тот путь, который был полем «Выход» при шифровании).

Если файл шифра был открыт в редакторе и сохранён как **UTF-8 с BOM**, в начале могли добавиться 3 байта `EF BB BF` — при расшифровке BOM в начале файла **автоматически удаляется** в коде сервиса.

## 5. Форматы данных

- **Блок:** 8 байт (внутри — список целых 0…255).
- **Ключ:** 8 байт = **16 hex** (без префикса `0x`).
- **Текст в GUI:** UTF-8; к блокам применяется **PKCS#7** (как в `utf8_to_blocks` и при шифровании файла).
- **Hex шифра:** непрерывная строка; длина чётная; число байт для расшифровки текста — **кратно 8**.

После **шифрования файла** длина файла шифра **всегда кратна 8** (цепочка блоков по 8 байт).

## 6. Криптоанализ: модель данных

### 6.1. `AttackContext` (вход атаки)

- `num_rounds` — число раундов.
- `plaintext_blocks` / `ciphertext_blocks` — блоки из полей P и C.
- `plain_pairs` / `cipher_pairs` — дополнительные пары блоков (например P2/C2), если заданы.
- `known_key_bytes` — частично известные байты ключа (если используется).

### 6.2. `AnalysisResult` (выход атаки)

- `success`, `message`, `elapsed_seconds`.
- `recovered_key_hex` — при успешном восстановлении ключа.
- `details` — многострочные пояснения в лог.
- `metadata` — словарь; для графиков используется, например, число **`candidates`**.

### 6.3. Реестр атак (`analysis/registry.py`, `default_registry`)

Порядок и состав могут меняться в коде; типичный набор:

1. Перебор равномерного ключа (8× один байт).
2. Ограниченный полный перебор (лимит кандидатов настраивается в GUI).
3. Известный открытый текст.
4. Линейный криптоанализ (при необходимости — оракул шифрования).
5. Дифференциальный анализ.
6. Встреча посередине (демо).
7. Интегральный анализ.
8. Выбранный открытый текст (демо).
9. Статистический анализ.

Требования каждой атаки к полям контекста смотрите в её реализации и в сообщениях об ошибках в логе.

---

## 7. Графики (`gui/attack_charts.py`)

1. **Время** — длительность каждой атаки (при большом разбросе ось времени может быть логарифмической).
2. **Успех и кандидаты** — для каждого метода: слева цветовая полоса успеха (зелёный / серый), справа оранжевая полоса, длина которой отражает нормированное число кандидатов; подписи `n=…` при больших числах сокращаются.
3. **Гистограмма** — частоты байт 0…255 по данным hex из поля C.

Поля и размеры шрифтов подстраиваются под размер виджета; после отрисовки выполняется дополнительная подгонка полей, чтобы подписи не обрезались.

## 8. Разработчикам

### 8.1. Добавление новой атаки

1. Реализуйте класс с полями `id`, `title` и методом `run(self, ctx: AttackContext) -> AnalysisResult`, соответствующий `KeyRecoveryAttack` в `analysis/protocols.py`.
2. Зарегистрируйте экземпляр в `default_registry()` в `registry.py` (передайте зависимости через `CryptoAnalysisDependencies`, если нужен движок/расписание).
3. При необходимости заполните `metadata["candidates"]` для отображения на втором графике.

### 8.2. Зависимости GUI

- `matplotlib` с backend для Qt (`FigureCanvasQTAgg`) — для виджета графиков.
- Без matplotlib вкладка анализа покажет текстовую подсказку вместо диаграмм.

### 8.3. Точки входа

- `python main.py` / `python run_gui.py` → `safer_k64.gui.bootstrap.main()`.
- Программный запуск окна: создать `QApplication`, затем `MainWindow` из `safer_k64.gui.main_window`.

---

## 9. Ограничения и замечания

- Шифр **учебный**; стойкость не сравнима с современными стандартами (AES и т.д.).
- Атаки часто **демонстрационные** (ограничения по данным, числу ключей, раундам).
- Расшифрование файла даёт осмысленный результат только при **том же ключе и числе раундов**, что при шифровании, и при **корректном файле шифра**.
